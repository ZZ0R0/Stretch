## Protocole 1 — Validation du RPE (P0)

### Objectif
Vérifier que le RPE résout la saturation du reward en mode normal et ne dégrade pas le mode inversé.

### Pré-requis
- RPE implémenté (RPE-1 à RPE-5)
- Backend CPU activé pour validation de référence

### 1.1 Test RPE convergence (CPU, déterministe)

**Config** : `config_v5.2_rpe_test.toml`
```toml
[reward]
rpe_enabled = true
rpe_alpha = 0.05

[v5_task]
task_mode = "Symmetric"
baseline_mode = "FullLearning"

[compute]
backend = "cpu"

[simulation]
total_ticks = 10000
```

**Vérifications** :
1. $\bar{r}(0) = 0$ (init)
2. $\bar{r}(t)$ converge vers $\approx 2p - 1$ (vérifier sur les métriques exportées)
3. $\mathbb{E}[\delta]$ sur les 1000 derniers ticks $< 0.1$ en valeur absolue
4. $\sigma(\delta)$ mesurable et > 0

**Critère de réussite** : $\bar{r}$ atteint un plateau stable. $\delta$ oscille autour de 0.

### 1.2 Matrice 2×2 avec RPE (GPU)

**Configs** : 4 fichiers config_v5.2_*

| Config | Task mode | Baseline | Mapping |
|---|---|---|---|
| `config_v5.2_sym_normal_full.toml` | Symmetric | FullLearning | Normal |
| `config_v5.2_sym_normal_topo.toml` | Symmetric | TopologyOnly | Normal |
| `config_v5.2_sym_inv_full.toml` | Symmetric | FullLearning | Inverted |
| `config_v5.2_sym_inv_topo.toml` | Symmetric | TopologyOnly | Inverted |

**Paramètres communs** :
- `domain.size = 50000`, `domain.topology = "knn_3d"`, `domain.k_neighbors = 12`
- `simulation.total_ticks = 10000`
- `reward.rpe_enabled = true`, `reward.rpe_alpha = 0.05`
- `compute.backend = "gpu"`

**Résultat attendu** :

| | TopologyOnly | FullLearning | Δ | Gate |
|---|---|---|---|---|
| Normal | ~97% | **≥ 97%** | **≥ 0** | ✅ |
| Inversé | ~3% | **≥ 80%** | **≥ +77** | ✅ |

**Critère de réussite** : les deux Δ sont ≥ 0. Le mode normal n'est plus dégradé.

### 1.3 Balayage de α

Tester 5 valeurs de $\alpha$ pour identifier l'optimum :

| α | Comportement attendu |
|---|---|
| 0.001 | Baseline très lente, δ quasi = r pendant longtemps |
| 0.01 | Baseline lente, convergence en ~100 ticks |
| 0.05 | **Nominal** — convergence en ~20 ticks |
| 0.10 | Baseline rapide, potentiellement bruité |
| 0.50 | Très rapide, δ oscillant fortement |

Exécuter la matrice 2×2 pour chaque α. Reporter accuracy et Δ.

## Protocole 2 — Validation GPU V5 sustained (P0)

### Objectif
Vérifier que les fonctionnalités V5 sustained (decay adaptatif, réverbération, reset policy) produisent les mêmes résultats sur GPU et CPU.

### 2.1 Test de cohérence CPU ↔ GPU

**Config** : `config_v5.2_sustained_test.toml`
```toml
[v5_sustained]
adaptive_decay = true
k_local = 0.3
reverberation = true
reverb_gain = 0.12
reset_policy = "partial"

[simulation]
total_ticks = 1000
```

**Procédure** :
1. Exécuter avec `compute.backend = "cpu"` → enregistrer accuracy, énergie, conductance à tick 1000
2. Exécuter avec `compute.backend = "gpu"` → mêmes métriques
3. Comparer : écart < 0.5% sur chaque métrique

**Critère de réussite** : accuracy CPU ≡ accuracy GPU (±1 essai sur 30~35 trials).

### 2.2 Test comparatif sustained ON/OFF (GPU)

| Config | Sustained | Attendu |
|---|---|---|
| V5.2 sustained OFF | `adaptive_decay=false, reverberation=false, reset_policy="full"` | Résultats ≡ V5.1 |
| V5.2 sustained ON | `adaptive_decay=true, reverberation=true, reset_policy="partial"` | Sustain ratio > 0.1 |

**Critère de réussite** : sustained OFF reproduit les résultats V5.1 exactement. Sustained ON augmente le sustain ratio.

### 2.3 Test mémoire et performance GPU

Mesurer sur la config 50k nœuds :

| Métrique | V5.1 | V5.2 | Max delta |
|---|---|---|---|
| Mémoire GPU | ~25 MB | mesurer | < +3 MB |
| Tick time (ms) | mesurer V5.1 | mesurer V5.2 | < +15% |

**Procédure** : `run_profiled_tick()` sur 100 ticks, reporter min/avg/max par phase.

## Protocole 4 — Remap avec oubli accéléré (P1)

### Objectif
Vérifier que le remap atteint >70% en phase 2 grâce au RPE + oubli accéléré.

### 4.1 Test Remap baseline (RPE seul)

**Config** : `config_v5.2_remap_rpe.toml`
```toml
[v5_task]
task_mode = "Remap"
remap_at_tick = 5000

[reward]
rpe_enabled = true
rpe_alpha = 0.05
accelerated_forgetting = false

[simulation]
total_ticks = 10000
```

**Mesure** : accuracy phase 1 (ticks 0–5000) et phase 2 (ticks 5000–10000) séparément.

Si phase 2 > 70% : l'oubli accéléré n'est pas nécessaire. Passer au protocole 5.

### 4.2 Test Remap avec oubli accéléré

**Config** : `config_v5.2_remap_forget.toml`
```toml
[reward]
rpe_enabled = true
rpe_alpha = 0.05
accelerated_forgetting = true
rho_boost = 0.01

[v5_task]
task_mode = "Remap"
remap_at_tick = 5000
```

**Résultat attendu** :

| Phase | Ticks | Accuracy cible |
|---|---|---|
| Phase 1 | 0 – 5000 | > 90% |
| Phase 2 | 5000 – 10000 | **> 70%** |

### 4.3 Balayage ρ_boost

Si 4.2 échoue, tester $\rho_{\text{boost}} \in \{0.005, 0.01, 0.02, 0.05\}$.

## Protocole 5 — Multi-seed (P1)

### Objectif
Quantifier la variance inter-exécutions et fournir des résultats robustes $\mu \pm \sigma$.

### 5.1 Procédure

Exécuter la matrice 2×2 (4 configs × 2 mappings) avec 5 seeds différentes :
- Seeds : 42, 123, 456, 789, 1337

Pour chaque condition, reporter :
- accuracy: $\mu \pm \sigma$
- Δ : $\mu \pm \sigma$
- Temps de convergence (tick auquel accuracy > 80% pour la première fois)

### 5.2 Critère de robustesse

| Métrique | Cible |
|---|---|
| $\sigma(\text{accuracy})$ par condition | < 5 pp |
| $\sigma(\Delta)$ | < 10 pp |
| Toutes les seeds passent la gate §4.1 du CdC | ✅ requis |

## Protocole 6 — Profiling GPU V5.2 (P1)

### Objectif
Mesurer l'overhead des nouvelles passes GPU et vérifier la contrainte PERF-1 (<15%).

### 6.1 Benchmark comparatif V5.1 vs V5.2

**Procédure** :
1. Compiler V5.1 (git stash ou branche) → `cargo build --release`
2. Exécuter 100 ticks sur la config 50k nœuds avec profiling GPU timestamp
3. Compiler V5.2 → `cargo build --release`
4. Exécuter les mêmes 100 ticks avec profiling

**Métriques** :
- Temps total par tick (ms) : min, avg, max
- Temps par phase (les 12 phases V5.1 vs 14 phases V5.2)
- Overhead V5.2 = (avg_V5.2 - avg_V5.1) / avg_V5.1

**Critère** : overhead < 15%.

### 6.2 Détail des nouvelles phases

| Phase | Threads | Lectures | Écritures | Estimation |
|---|---|---|---|---|
| reverberation | N nœuds | 1 float (prev) + 1 Node | 1 Node | ~0.05 ms |
| adaptive_decay | N nœuds | CSR outgoing (variable) + 1 Node | 1 Node | ~0.1 ms |
| snapshot_activations | N nœuds | 1 Node activation | 1 float | ~0.03 ms |

Estimation totale : ~0.2 ms d'overhead sur ~2–3 ms par tick → ~7–10%.
