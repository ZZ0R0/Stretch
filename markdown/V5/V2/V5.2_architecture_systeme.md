## 1. Vue d'ensemble — Pipeline V5.2 GPU

### 1.1 Pipeline complet (17 passes, single submit)

```
┌─────────────────────────────────────────────────────────┐
│ CommandEncoder V5.2 (1 seul submit par tick)            │
├─────────────────────────────────────────────────────────┤
│  1. reset_activations     ← modifié: reset policy       │
│  2. inject                                              │
│  3. zone_measure                                        │
│  4. zone_pid                                            │
│  5. zone_apply                                          │
│  6. source_contribs                                     │
│  7. propagation                                         │
│  8. apply_influences      ← renommé (ex apply_dissipate)│
│  9. ★ reverberation       ← NOUVEAU                     │
│ 10. ★ adaptive_decay      ← NOUVEAU                     │
│ 11. dissipation           ← simplifié (decay seulement) │
│ 12. plasticity            ← modifié: δ au lieu de r     │
│ 13. budget_sum                                          │
│ 14. budget_scale                                        │
│ 15. sync_conductances                                   │
│ 16. readout               ← modifié: num_classes param  │
│ 17. metrics_nodes + metrics_edges                       │
└─────────────────────────────────────────────────────────┘
                        ↓
              1 readback : GpuMetricsRaw (64 bytes)
              + readout_scores (num_classes × 4 bytes)
              ↓ CPU-side post-processing :
              decision = argmax(scores)
              reward = f(decision, target)
              δ = reward - r̄      ← RPE (CPU)
              r̄ += α × (reward - r̄)
              dopamine.update(δ)   ← δ remplace r
```

### 1.2 Différences V5.1 → V5.2

| # | Passe V5.1 | Passe V5.2 | Changement |
|---|---|---|---|
| 1 | reset_activations (full) | reset_activations (configurable) | Ajout du champ `reset_policy` dans GpuParams |
| 8 | apply_and_dissipate (fusionné) | apply_influences (influences seulement) | Séparation : influences et dissipation deviennent des passes distinctes |
| 9 | — | **reverberation** (nouveau) | $\phi_i += r_{\text{local}} \cdot \phi_{\text{prev},i}$ |
| 10 | — | **adaptive_decay** (nouveau) | $\alpha_{\text{eff}} = \alpha \cdot (1 - k \cdot \bar{a}_{\text{voisins}})$ |
| 11 | (dans apply_and_dissipate) | dissipation (standalone) | Decay standard + jitter, dans sa propre passe |
| 12 | plasticity (dopamine = d_total) | plasticity (dopamine = δ_RPE) | Le champ `dopamine` de GpuParams contient $\delta(t)$ au lieu de $d_{\text{total}}$ |
| 16 | readout (num_classes = 2) | readout (num_classes = param) | Lecture depuis GpuParams |

```rust
pub struct RewardSystem {
    pub current: f32,
    pub cumulative: f32,
    pub count: usize,
    // V5.2 RPE
    pub baseline: f32,       // r̄(t) — moyenne glissante
    pub rpe_delta: f32,      // δ(t) = r(t) - r̄(t)
}

impl RewardSystem {
    /// V5.2 : Calculer le RPE et mettre à jour la baseline.
    pub fn compute_rpe(&mut self, alpha: f32) {
        self.rpe_delta = self.current - self.baseline;
        self.baseline += alpha * (self.current - self.baseline);
    }
}
```

**Modifications dans `step_gpu()`** :

```
Avant (V5.1) :

  2. gpu.run_full_tick()
```

### 2.6 Nouveaux shaders WGSL

#### `reverberation.wgsl` (~30 lignes)

```wgsl
// 1 thread par nœud
// Lit prev_activations[gid], ajoute reverb_gain × prev au nœud
@group(0) @binding(0) var<storage, read_write> nodes: array<GpuNode>;
@group(0) @binding(1) var<storage, read> prev_activations: array<f32>;
@group(0) @binding(2) var<uniform> params: GpuParams;

@compute @workgroup_size(256)
fn reverberation(@builtin(global_invocation_id) gid: vec3<u32>) {
    let idx = gid.x;
    if idx >= params.num_nodes { return; }
    let prev = prev_activations[idx];
    let reverb = params.reverb_gain * prev;
    if reverb > 0.001 {
        nodes[idx].activation = min(nodes[idx].activation + reverb, 5.0);
    }
}
```

#### `adaptive_decay.wgsl` (~60 lignes)

```wgsl
// 1 thread par nœud
// Calcule la moyenne d'activité des voisins sortants via le CSR outgoing
// Puis applique α_eff = α_base × (1 - k_local × mean_neighbor)
// NOTE : nécessite le CSR outgoing uploadé sur GPU

@group(0) @binding(0) var<storage, read_write> nodes: array<GpuNode>;
@group(0) @binding(1) var<storage, read> out_offsets: array<u32>;
@group(0) @binding(2) var<storage, read> out_targets: array<u32>;
@group(0) @binding(3) var<uniform> params: GpuParams;

@compute @workgroup_size(256)
fn adaptive_decay(@builtin(global_invocation_id) gid: vec3<u32>) {
    let idx = gid.x;
    if idx >= params.num_nodes { return; }

    let act = nodes[idx].activation;
    if act <= params.activation_min { return; }

    // Moyenne d'activité des voisins sortants
    let start = out_offsets[idx];
    let end = out_offsets[idx + 1u];
    var sum_act = 0.0;
    var count = 0u;
    for (var k = start; k < end; k = k + 1u) {
        let target = out_targets[k];
        sum_act += nodes[target].activation;
        count += 1u;
    }
    let mean_act = select(0.0, sum_act / f32(count), count > 0u);

    // α_eff = α_base × clamp(1 - k × mean_act, 0.2, 1.0)
    let factor = clamp(1.0 - params.k_local * mean_act, 0.2, 1.0);
    let effective_decay = params.activation_decay * factor;

    nodes[idx].activation = max(act * (1.0 - effective_decay), params.activation_min);
}
```

```wgsl
fn reset_activations(@builtin(global_invocation_id) gid: vec3<u32>) {
    let idx = gid.x;
    if idx >= params.num_nodes { return; }

    switch params.reset_policy {
        case 0u: { // full
            nodes[idx].activation = 0.0;

        case 1u: { // partial — garde 10%
            nodes[idx].activation *= 0.1;
        }
        case 2u: { // none
            // rien
        }
        default: {
            nodes[idx].activation = 0.0;
        }
    }
}
```

## 3. Buffers GPU — Bilan mémoire

### 3.1 Nouveaux buffers

| Buffer | Taille (50k nœuds) | Usage |
|---|---|---|
| `prev_activations_buf` | 4B × 50k = 200 KB | Copie des activations du tick précédent (réverbération) |
| `out_offsets_buf` | 4B × (50k+1) = 200 KB | CSR outgoing offsets (decay adaptatif) |
| `out_targets_buf` | 4B × 500k = 2 MB | CSR outgoing target nodes (decay adaptatif) |

### 3.2 Impact mémoire total

| Composant | V5.1 | V5.2 | Delta |
|---|---|---|---|
| Buffers GPU | ~25 MB | ~27.4 MB | +2.4 MB |
| GpuParams | 192B | 224B | +32B |

L'overhead mémoire est négligeable (<10%).

### 3.3 Upload du CSR outgoing

Le CSR outgoing est actuellement construit dans `domain.rs` pour le chemin CPU mais **n'est pas uploadé sur GPU**. Le CSR incoming seul est uploadé (pour la propagation). Le decay adaptatif V5.2 nécessite le CSR outgoing sur GPU.

**Option A** : uploader le CSR outgoing complet → +2.2 MB. Simple, robuste.

**Option B** : réutiliser le CSR incoming avec une transformation source↔target. Non trivial, évité.

**Recommandation** : Option A. 2.2 MB est négligeable.

## 4. Schéma de données du RPE

```
                 ┌────────────────────────┐
                 │    GPU tick (17 passes) │
                 └──────────┬─────────────┘
                            │ readout_scores (32B)
                            ↓
                 ┌────────────────────────┐
                 │   CPU post-processing   │
                 │   1. decision = argmax  │
                 │   2. reward = ±1        │
                 │   3. δ = r - r̄          │  ← RPE
                 │   4. r̄ += α(r - r̄)     │
                 │   5. dopa.update(δ)     │
                 └──────────┬─────────────┘
                            │ δ(t) → GpuParams.dopamine
                            ↓              pour le tick t+1
                 ┌────────────────────────┐
                 │    GPU tick t+1         │
                 │    plasticity utilise δ  │
                 └────────────────────────┘
```

**Point clé** : le RPE est calculé côté CPU entre deux ticks GPU. Il n'y a **aucun readback supplémentaire** — le readout_scores est déjà lu à chaque tick pour la décision. Le RPE ajoute ~5 instructions CPU.

## 5. Restructuration du shader apply_and_dissipate

### 5.1 Situation V5.1

Le shader `apply_and_dissipate.wgsl` fusionne deux opérations :
1. Application des influences (résultat de la propagation)
2. Dissipation (decay + fatigue + inhibition + trace)

### 5.2 Plan V5.2

Séparer en 3 passes distinctes :

| Passe | Shader | Responsabilité |
|---|---|---|
| `apply_influences` | `apply_influences.wgsl` | `activation += influence; clamp(0, 10)` |
| `reverberation` | `reverberation.wgsl` | `activation += reverb_gain × prev_activation` |
| `adaptive_decay` + `dissipation` | `adaptive_decay.wgsl` | Decay adaptatif V5 + dissipation standard fusionnés |

**Alternative** : garder `apply_and_dissipate.wgsl` tel quel et ajouter les 2 passes V5 avant/après. Cette approche minimise les modifications mais complexifie l'ordre.

**Recommandation** : la séparation est plus propre et permet d'activer/désactiver chaque mécanisme indépendamment via des flags dans GpuParams.
